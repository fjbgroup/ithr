# IT System - One Page Instruction Flow

Use this flow as the quick guide for the IT Management System.

## 1. Sign In

1. Open **IT System** and sign in with your staff account.
2. If the system asks for a password change, update your password before continuing.
3. Check the notification bell for pending approvals, rejected requests, or system updates.

## 2. Start From The Dashboard

Use **Dashboard** to see the current inventory summary, pending work, recent activity, and quick links. Choose the module that matches your task:

| Task | Go To |
| --- | --- |
| Request hardware, software, access, or service support | **IT Request Form** |
| Check or manage IT assets | **IT Inventory > IT Assets** |
| Check or manage non-IT assets | **IT Inventory > Non-IT Assets** |
| Review asset write-off requests | **Write-off** |
| Track e-waste items | **E-Waste** |
| Track disposed items and proof records | **Disposal** |
| View exported inventory reports | **Reports** |
| Update your contact details, password, or signature | **Profile** |

## 3. Submit An IT Request

1. Go to **IT Request Form**.
2. Select the request type and fill in all required fields.
3. Attach supporting files if required.
4. Save as draft if the request is incomplete, or submit when ready.
5. Track the request from **IT Request Form** until it is approved, rejected, or returned for update.

## 4. Manage Assets

1. Go to **IT Inventory**.
2. Use search and filters to find the asset.
3. Open the asset record to review details, or use the available action buttons to add, edit, delete, write off, or move items to e-waste.
4. Requests that need approval appear under **Pending Requests** for authorized users.
5. Your own submitted changes appear under **My Requests**.

## 5. Approval Flow

Follow the approval action shown on screen:

1. **Staff** submits requests and monitors status.
2. **HOU** reviews assigned requests and approves or rejects them.
3. **Validator / IT Admin** validates technical details and finalizes IT actions.
4. **GM / CEO / Finance Admin** review write-off or finance-related items where required.
5. The system records each action in notifications and activity logs.

## 6. Write-Off, E-Waste, And Disposal

1. Use **Write-off** when an asset must be retired from active use.
2. After approval, route the item to **E-Waste** or **Disposal** based on the required process.
3. Update collection or disposal status when the physical handover is completed.
4. Keep proof records complete for audit and reporting.

## 7. Admin Maintenance

For IT Admin and Finance Admin roles:

1. Use **Users** to create, update, deactivate, or reset IT user accounts.
2. Use **Masterdata**, **Asset Classes**, **Brands**, and **Locations** to keep dropdown values accurate.
3. Use **Email Settings** to confirm notification delivery settings.
4. Use **Activity Logs** and **Reports** to review audit history and export records.

> [!TIP]
> If you are unsure where to begin, start at **Dashboard**, check notifications, then open the module named closest to your task.
